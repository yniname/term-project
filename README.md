一、程式功能與技術原理
1.本程式目的計算反應速率常數k，並動態呈現物質濃度反應情況。提供六個反應讓使用著挑選，並自行輸入溫度、時間、反應物濃度，計算出速率常數。利用數值積分法及odeint計算每時間步長的濃度更新，繪圖輸出。

二、使用方式:
1.執行Python.exe B14504061.py
2.從選單中挑選一個欲模擬的反應
3.輸入反應溫度、反應時間，及反應物的濃度
4.終端機上會輸出根據以上條件計算出的k
5.物質濃度變化圖上也會顯示k，若k極大或極小，會出現警告
6.模擬結束後，終端機會輸出轉化率及各物質最終濃度。
7.若是連鎖反應，則還會再輸出中間產物濃度極大值及達到極大值的時間

三、程式架構
1.開始
2.選擇反應
3.計算速率常數k
4.建立濃度計算模式
5.計算濃度更新
6.繪圖輸出

四、開發過程
1.階段一:先完成NO+O2-->NO2的濃度變化動態圖
(1)一開始直接利用公式解零級、一級、二級反應的濃度。但這個方法限制太多，通用性太低，而且生成物濃度圖明顯錯誤。採納Copilot建議改用數值積分法計算濃度，解(d[C])/dt=-k〖[C]〗^n，如此一來，後續也可以擴展至不同反應的計算。
2.階段二: 改寫成多反應方程式通用
(1)info_of_reaction:
用dictionary儲存反應式及Ea、A等資料，建立反應式選單，詢問並回傳使用者選擇的反應的資訊。
(2)concentration_update:
沿用原本的數值積分法，但把原本寫死的反應速率計算改成能用迴圈、變數去執行，能夠處理不同數量、係數的反應物、生成物。
(3)採納Copilot建議，將程式明確分工:
info_of reaction:負責回傳反應資訊。
concentration_update:計算濃度更新並儲存成陣列。
主程式:輸出反應式選單、畫圖。
(4)以不同反應溫度、物質濃度測試時，發現在某些時候輸出的圖急遽下降或上升，且出現非圓滑轉折處。進一步了解，發現用目前的迭代方法，僅能承受剛性方程，若遇到k值太大時(非剛性方程)，原本預設的dt會太大，導致濃度在一步之內急遽變化、數值發散、出現負濃度等情形，不符合實際情況。詢問Gemini解決方法，建議使用odeint解微分方程。新的計算方法，須先建立一個reaction model，定義dC/dt。並利用max(0, C[i])，避免出現負數情況。最後回傳濃度瞬時變化率。再來利用odeint積分求解，最後再確認濃度非負值，確保得到的結果符合實際情況。
(5) 由於在k很大的時候反應可能在短時間結束，因此圖形大部分為直線。為避免使用者以為出錯，畫布上會顯示警告語。K極小時亦然。
3.第三階段: 加入series reaction
(1)由於series action多了中間產物，有兩個Ea及A，因此建立新的function來儲存反應資料。
(2)series reaction濃度計算方式與前面一般反應不同，因此須建立新的reaction_model及concentration_update
(3)利用if、elif 區分使用者選擇的反應編號，執行一般反應計算或連鎖反應計算。
